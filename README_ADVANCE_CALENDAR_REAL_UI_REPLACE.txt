CWF ADVANCE CALENDAR REAL UI REPLACE
วันที่แก้: 2026-05-07

สาเหตุที่รอบก่อนเหมือนเดิม:
- ใน tech.html มีโค้ดปฏิทิน fallback เก่าแบบ inline อยู่ท้ายไฟล์
- โค้ด inline นี้ override ฟังก์ชันจาก app.js ทำให้ deploy แล้วหน้าแอพยังเปิด UI เก่าเหมือนเดิม

ไฟล์ที่แก้:
1) tech.html
- ลบโค้ดปฏิทิน fallback เก่าออก
- ให้ปุ่ม “ตั้งค่าปฏิทินรับงาน” เรียก UI ใหม่จาก app.js เท่านั้น

2) app.js
- เปลี่ยนปฏิทินรับงานล่วงหน้าเป็น UI ใหม่จริง
- เหลือแค่ 2 สถานะ: รับ / ไม่รับ
- โหมดปกติ: แตะวันที่เพื่อเปิด/ปิดรับงานล่วงหน้าทันที
- วันที่มีงานแล้วแสดง 📌 มีงาน และล็อกไม่ให้ปิดหรือแก้ไข
- มีปุ่ม ✏️ แก้ไขวันที่เลือก สำหรับตั้งเวลา จำนวนงาน จำนวนเครื่อง และหมายเหตุ
- วันที่มีค่าพิเศษแสดงไอคอน ⏰ ⚙️ 📝
- มีโหมดเลือกหลายวันสำหรับตั้งค่ารวม
- ตั้งค่าเหมือนเดือนก่อนยังอยู่
- งานด่วนยังไม่เกี่ยวกับปฏิทิน ใช้ปุ่มเปิด/ปิดรับงานหน้าแรกตามเดิม

3) index.js
- Backend จำกัดสถานะปฏิทินเหลือรับ/ไม่รับ
- ถ้าวันนั้นมีงานอยู่แล้ว API จะไม่ให้แก้วันนั้น
- Bulk save จะข้ามวันที่มีงานแล้ว และส่งจำนวนวันที่ข้ามกลับมา

4) sw.js
- bump cache version เพื่อให้ PWA โหลด app.js/tech.html ใหม่

วิธีติดตั้ง:
- แตก ZIP นี้
- นำไฟล์ app.js, index.js, tech.html, sw.js ไปทับที่ root ของ repo
- commit/push แล้วรอ Render deploy สำเร็จ
- หลัง deploy ให้ปิด PWA แล้วเปิดใหม่ หรือเข้า browser แล้ว hard refresh

Checklist ทดสอบ:
1. เปิดแอพช่าง
2. กดเมนูแฮมเบอร์เกอร์ > ตั้งค่าการรับงาน > ตั้งค่าปฏิทินรับงาน
3. ต้องเห็น UI ใหม่ ไม่ใช่ UI เดิม
4. แตะวันที่ว่าง 1 ครั้ง = เปิด/ปิดรับงานล่วงหน้าทันที
5. กดแก้ไขวันที่เลือก ตั้งเวลา/จำนวนงาน/จำนวนเครื่อง/หมายเหตุ แล้วบันทึก
6. วันนั้นต้องมีไอคอน ⏰ ⚙️ 📝 ตามค่าที่ตั้ง
7. วันที่มีงานแล้วต้องแสดง 📌 มีงาน และกดปิด/แก้ไม่ได้
8. งานด่วนหน้าแรกยังใช้ปุ่มเปิด/ปิดรับงานเหมือนเดิม
