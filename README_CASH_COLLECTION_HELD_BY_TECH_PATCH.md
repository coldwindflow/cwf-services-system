# CWF Patch: เงินสดที่ช่างเก็บไว้ แล้วหักในบัญชีงวดจ่าย

## ปัญหา
ลูกค้าจ่ายเงินสดให้ช่างหน้างาน แต่ช่างไม่ได้โอนเงินเข้าบริษัททันที และบริษัทต้องการให้ช่างเก็บเงินสดนั้นไว้ก่อน แล้วนำยอดไปหักออกจากยอดจ่ายช่างในบัญชีงวดถัดไป/งวดที่เกี่ยวข้อง

## แนวทางที่แก้
ไม่ใช้ตารางเงินประกัน เพราะเงินสดจากลูกค้าเป็นเงินรับแทนบริษัท ไม่ใช่เงินประกันของช่าง

เพิ่ม ledger แยกใหม่ชื่อ `technician_cash_collections` สำหรับบันทึก “เงินสดที่ช่างถือไว้” และเชื่อมกับระบบจ่ายช่างด้วย `technician_payout_adjustments` แบบยอดติดลบ

ตัวอย่าง:
- รายได้ช่างงวดนี้: 2,000 บาท
- ลูกค้าจ่ายเงินสดให้ช่างถือไว้: 1,500 บาท
- ระบบเพิ่ม adjustment: -1,500 บาท
- ยอดที่บริษัทต้องโอนให้ช่างจริง: 500 บาท

ถ้าเงินสดที่ช่างถือไว้มากกว่ายอดรายได้งวดนั้น ระบบจะแสดงยอดสุทธิติดลบ/คงเหลือ 0 ในหน้าจ่าย เพื่อให้บัญชีเห็นว่าต้องตรวจยอดคงค้างหรือให้ช่างคืน/ยกยอดไปงวดถัดไปตามนโยบายบริษัท

## ไฟล์ที่เพิ่ม/แก้
- `server/services/technicianCashCollections.js` เพิ่ม service แยก ไม่ยัด logic เข้าหน้า Index ทั้งหมด
- `index.js` เชื่อม service กับ flow ปิดงาน, บันทึกรับเงิน, schema, และ endpoint sync manual
- `server/routes/accountingReadOnly.js` เพิ่มข้อมูลเงินสดที่ช่างถือไว้ในหน้ารายรับ
- `admin-accounting-v2.js` แสดง badge/ปุ่มหักเงินสดในหน้า Accounting และแสดงยอดเงินสดที่หักในหน้าจ่ายช่าง

## วิธีใช้งานหลังลง patch
1. ตอนช่างปิดงาน ให้เลือกวิธีชำระเงินเป็น `cash_to_technician`
2. ใส่จำนวนเงินสดที่รับ และติ๊กยืนยันรับเงินสด
3. ระบบจะสร้างรายการ `technician_cash_collections`
4. ระบบจะสร้าง adjustment ติดลบในงวดจ่ายของช่างคนนั้นให้อัตโนมัติ
5. ฝั่ง Admin Accounting > รายรับ จะเห็น badge “เงินสดอยู่กับช่าง/หักเงินสดช่างแล้ว”
6. ฝั่ง Admin Accounting > จ่ายเงินช่าง จะเห็นยอด “ปรับยอด” และ “เงินสดช่างถือไว้” ถูกหักออกจากยอดสุทธิ

## กรณีงานวันนี้ปิดไปแล้วก่อนลง patch
หลังลง patch ให้เข้า Admin Accounting > รายรับ แล้วกดปุ่ม “หักเงินสดจากบัญชีช่าง” ในรายการงานนั้น ระบบจะ sync ยอดเงินสดที่ช่างถือไว้เข้า ledger และ adjustment ให้ย้อนหลังแบบ audit

## API ที่เพิ่ม
`POST /admin/accounting/revenue/:job_id/sync-tech-cash`

ใช้สำหรับบังคับ sync งานที่มี `close_payment_method = cash_to_technician` และ `close_cash_confirmed = true` ให้กลายเป็น adjustment หักยอดช่าง

## การทดสอบ
```bash
node --check index.js
node --check server/routes/accountingReadOnly.js
node --check server/services/technicianCashCollections.js
node --check admin-accounting-v2.js
```

## ผลลัพธ์ที่ควรเห็น
- ปิดงานด้วยเงินสดที่ช่างรับไว้แล้วไม่ทำให้ระบบบัญชีเงินสด/เงินประกันปนกัน
- ยอดจ่ายช่างถูกหักด้วย adjustment แบบ audit ได้
- Admin เห็นงานที่เป็นเงินสดช่างถือไว้จากหน้ารายรับ
- Admin กด sync ย้อนหลังได้ถ้างานปิดไปก่อน patch
