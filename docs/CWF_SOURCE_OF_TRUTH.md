# CWF Source of Truth

เอกสารนี้กำหนดกติกาหลักที่ AI/Codex ต้องยึดก่อนแก้ระบบ CWF

## งานแก้ไข / Revisit Flow

งานแก้ไขเป็น flow แยกจากงานปกติ

- ไม่มีค่าบริการที่ต้องเก็บจากลูกค้า
- ไม่มีค่าตอบแทนเพิ่มเติมให้ช่าง
- ไม่บังคับรูปเนมเพลท เพราะมีจากงานเดิมแล้ว
- ต้องมีรูปก่อนแก้ไข
- ต้องมีรูปสาเหตุ/จุดปัญหา
- ต้องมีรูปหลังแก้ไข
- ต้องเลือกว่าสาเหตุเกิดจากใคร พร้อมกรอกเหตุผล
- ต้องเลือกผลการแก้ไขและกรอกสรุปผลงานแก้ไข
- ลายเซ็นปิดงานเป็นลายเซ็นช่าง

## กติกานัดหมายงานแก้ไข

ช่างต้องบันทึกเฉพาะวันและเวลาที่ตกลงกับลูกค้า ไม่ต้องบังคับให้พิมพ์สรุปการคุย

เวลานัดแก้ไขต้องไม่ชนกับงานปัจจุบันหรืองานล่วงหน้าที่ช่างมีอยู่แล้ว

ถ้าเวลาที่เลือกชนกับงานอื่น:

- ระบบต้องไม่บันทึกเวลานัดแก้ไข
- ระบบต้องไม่เปลี่ยน `appointment_datetime`
- ระบบต้องแสดง popup ภาษาไทยให้ช่างติดต่อแอดมินเพื่อย้ายคิวงานเดิม หรือเลือกเวลาใหม่
- ไม่ต้องมีปุ่มโทรหาแอดมิน
- ช่างห้าม override คิวชนเอง

ข้อความ popup:

```text
เวลานี้ชนกับงานอื่นที่มีอยู่แล้ว

หากจำเป็นต้องเข้าช่วงเวลานี้จริง ๆ
กรุณาติดต่อแอดมินเพื่อย้ายคิวงานเดิมก่อน
หรือเลือกเวลานัดแก้ไขใหม่
```

## การเช็คคิวชน

การเช็คคิวชนต้องทำที่ backend เป็นหลัก

- ต้องตรวจงานที่ assign ให้ช่างคนเดียวกัน
- ต้องรวมงานปัจจุบันและงานล่วงหน้า
- ต้อง exclude job_id ของงานแก้ไขใบปัจจุบัน
- ถ้าเวลาชน ต้องตอบ `REVISIT_SCHEDULE_CONFLICT`
- frontend มีหน้าที่แสดง popup เท่านั้น
