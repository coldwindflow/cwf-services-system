CWF Advance Work Calendar Toggle/Lock Fix

ไฟล์นี้เป็นแพตช์เฉพาะระบบ “ตั้งค่าปฏิทินรับงานล่วงหน้า” ฝั่งช่าง

สิ่งที่แก้:
1. ลดสถานะเหลือ 2 แบบเท่านั้น
   - ✅ รับงานล่วงหน้า
   - ❌ ไม่รับงานล่วงหน้า
   วันหยุด/วันลา/ไม่ว่าง ให้ใส่เป็นหมายเหตุแทน

2. แตะวันที่เพื่อเปิด/ปิดทันที
   - วันที่ที่ยังไม่รับ: แตะแล้วเปิดรับงานล่วงหน้า
   - วันที่ที่รับอยู่: แตะแล้วปิดรับงานล่วงหน้า
   - ไม่ต้องกดบันทึกสำหรับการเปิด/ปิดธรรมดา

3. แก้ไขวันที่เลือกได้
   - กด “✏️ แก้ไขวันที่เลือก” เพื่อกำหนดค่าเฉพาะวันนั้น
   - เวลาเริ่ม
   - เวลาเลิก
   - จำนวนงานสูงสุดต่อวัน
   - จำนวนเครื่องสูงสุดต่อวัน
   - หมายเหตุถึงแอดมิน

4. วันที่มีค่าพิเศษดูออกทันที
   - ⏰ = เวลาไม่ใช่ค่าเริ่มต้น 09:00–18:00
   - ⚙️ = จำนวนงาน/จำนวนเครื่องไม่ใช่ค่าเริ่มต้น 1 งาน / 5 เครื่อง
   - 📝 = มีหมายเหตุถึงแอดมิน

5. วันที่มีงานแล้วล็อกไม่ให้แก้
   - แสดง 📌 มีงาน
   - ห้ามปิดรับงานหรือแก้ไขวันนั้น
   - ถ้ากดจะแจ้ง popup ให้ติดต่อแอดมิน
   - backend กันซ้ำด้วย HTTP 409

6. เลือกหลายวันยังใช้ได้
   - ใช้สำหรับตั้งค่ารวมหลายวัน
   - ระบบข้ามวันที่มีงานอยู่แล้วและแจ้งจำนวนวันที่ข้าม

7. เพิ่มปุ่ม “📋 ตั้งค่าเหมือนเดือนก่อน”
   - คัดลอกสถานะ เวลา จำนวนงาน จำนวนเครื่อง และหมายเหตุจากเดือนก่อน
   - ข้ามวันที่มีงานอยู่แล้ว

ไฟล์ที่แก้:
- app.js
- index.js
- sw.js

วิธีติดตั้ง:
1. แตก ZIP นี้
2. นำไฟล์ app.js, index.js, sw.js ไปทับที่ root repo
3. commit/push ขึ้น GitHub
4. รอ Render deploy สำเร็จ
5. ปิด PWA แล้วเปิดใหม่ หรือ refresh/l้าง cache เพื่อให้ service worker โหลด v49

Checklist ทดสอบ:
- เปิดแอพช่าง > เมนูแฮมเบอร์เกอร์ > ตั้งค่าการรับงาน > ตั้งค่าปฏิทินรับงาน
- แตะวันที่ว่างแล้วต้องเปิดรับทันที
- แตะวันเดิมอีกครั้งแล้วต้องปิดรับทันที
- กดแก้ไขวันที่เลือก ใส่เวลา 12:00–18:00 และหมายเหตุ แล้วบันทึก
- วันที่นั้นต้องมีไอคอน ⏰/📝 แสดงบนช่องวันที่
- วันที่มีงานอยู่แล้วต้องแสดง 📌 และกดปิด/แก้ไขไม่ได้
- ทดลองเลือกหลายวัน แล้วตั้งเป็นรับ/ไม่รับ ระบบต้องข้ามวันที่มีงานอยู่แล้ว
- ปุ่มรับงานด่วนหน้าแรกต้องยังใช้งานตามเดิม และไม่เกี่ยวกับปฏิทินนี้
