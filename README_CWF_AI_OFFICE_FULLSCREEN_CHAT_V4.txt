CWF AI Office Fullscreen Chat v4 Brain Fix

Goal
- Keep the office/game scene as the AI selector.
- Open a full-screen ChatGPT-style chat after choosing an AI character.
- Remove clutter: no quick command grid and no separate phone-search button.
- Add conversation memory for the current browser session so follow-up questions can use recent context.
- Keep Phase 1 read-only: no job create/edit/delete/status/payment/LINE send actions.

Files
- admin-ai-office.html
- admin-ai-office.js
- tools/apply-ai-office-fullscreen-chat-v4.sh

What changed from v3
1. Frontend now sends conversation_history with the latest 12 user/AI messages to /admin/ai-office/ask.
2. Browser session keeps per-agent chat history in sessionStorage.
3. Backend patch adds sanitizeConversationHistory() and passes safe history into the grounded AI prompt.
4. Backend prompt now behaves like CWF Operations Copilot: Thai, professional, direct, contextual, grounded in real JSON data only.
5. Extra AI Office PIN is disabled after admin login. requireAdminSession remains the protection layer.
6. No quick command UI. No separate phone search UI. Ask naturally in chat, e.g. “ค้นงานจากเบอร์ 098...”.

Install
From the repository root after extracting this ZIP:

bash tools/apply-ai-office-fullscreen-chat-v4.sh .

Then deploy and hard refresh /admin/ai-office.

Manual test checklist
- Login as admin.
- Open /admin/ai-office without extra PIN.
- Pick Admin AI and ask: วันนี้มีงานอะไรบ้าง
- Follow up: งานไหนเสี่ยงสุด
- Follow up: ร่างข้อความแจ้งลูกค้าคนนั้นให้หน่อย
- Ask: ค้นงานจากเบอร์ 098xxxxxxx
- Switch to Ops AI and ask: งานไหนยังไม่ปิด
- Check mobile: chat is full screen, readable, send button small, input fixed at bottom.
- Check desktop: office scene and full-screen chat both usable.
- Confirm no write actions were added.

Rollback
- Restore backup files generated by the script:
  admin-ai-office.html.bak.<timestamp>
  admin-ai-office.js.bak.<timestamp>
- Or revert the deployment commit.
